I will create a Dockerfile named `codex.dockerfile` that:

- Uses the official Python 3.12 slim-bullseye base image (to align with recent Python version support and smaller size).
- Installs git and system dependencies needed by Python packages.
- Copies the repository content to `/app`.
- Sets `/app` as the working directory.
- Installs the repository in editable mode to allow development changes using `pip install -e .`.
- Defaults to starting an interactive bash shell.

This will give you a CLI into the container with the repo installed at the root working directory.

FROM python:3.12-slim-bullseye

# Install git and system dependencies
RUN apt-get update && apt-get install -y --no-install-recommends \
    git \
    build-essential \
    && rm -rf /var/lib/apt/lists/*

# Set working directory
WORKDIR /app

# Copy the entire repo into the container
COPY . /app

# Install the repo in editable mode (dev mode)
RUN pip install --no-cache-dir -e .

# Default command: bash shell
CMD ["/bin/bash"]
